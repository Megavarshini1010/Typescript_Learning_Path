**# Day 01 - TypeScript Interview Questions**



**## 1. What is TypeScript?**



**\*\*Answer:\*\***



**TypeScript is an open-source programming language developed by Microsoft. It is a superset of JavaScript that adds static typing and compiles into JavaScript.**



**---**



**## 2. What is a superset?**



**\*\*Answer:\*\***



**A superset is a language that includes all the features of another language while adding new features.**



**TypeScript contains all JavaScript features plus additional features like static typing, interfaces, enums, and generics.**



**---**



**## 3. Why do we use TypeScript?**



**\*\*Answer:\*\***



**- Static Typing**

**- Better IntelliSense**

**- Compile-Time Error Detection**

**- Easier Maintenance**

**- Better Code Quality**

**- Better Refactoring**

**- Suitable for Large Applications**





**## 4. What is Static Typing?**



**\*\*Answer:\*\***



**Static typing means the data type of a variable is checked during development before the program executes.**



**Example:**



**```ts**

**let age: number = 25;**

**```**





**## 5. Does TypeScript run directly in the browser?**



**\*\*Answer:\*\***



**No.**



**TypeScript is first compiled into JavaScript using the TypeScript Compiler (tsc). The generated JavaScript then runs in the browser or Node.js.**





**## 6. Difference between JavaScript and TypeScript?**



**| JavaScript | TypeScript |**

**|------------|------------|**

**| Dynamic Typing | Static Typing |**

**| Runs directly | Compiles to JavaScript |**

**| Runtime Errors | Compile-Time Errors |**

**| No Type Checking | Type Checking |**

**| Good for Small Projects | Good for Large Projects |**





**## 7. What is the TypeScript Compiler?**



**\*\*Answer:\*\***



**The TypeScript Compiler (tsc) converts TypeScript (.ts) files into JavaScript (.js) files.**



**Example:**



**```**

**app.ts**



**↓**



**tsc**



**↓**



**app.js**

**```**



**---**



**## 8. Can JavaScript code run in TypeScript?**



**\*\*Answer:\*\***



**Yes.**



**Since TypeScript is a superset of JavaScript, every valid JavaScript program is also valid TypeScript.**

**## 12. What is the difference between compile-time and runtime errors?**



**\*\*Compile-Time Errors\*\***

**- Found while writing or compiling the code.**

**- Detected by the TypeScript compiler.**

**- Example: Assigning a string to a number variable.**



**\*\*Runtime Errors\*\***

**- Found when the application is running.**

**- Not detected by the compiler.**

**- Example: Accessing a property of `undefined`.**
